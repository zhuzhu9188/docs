一个网站能不能留住访客，首页前三秒就决定了。用户不会因为你的代码写得漂亮而留下来，他们只关心页面能不能让他们快速看懂"这个网站是干什么的，对我有没有用"。本文不谈纯粹的美学理论，只聚焦于那些经过验证、能直接影响转化率的设计决策。

## 页面设计工作流

按照以下步骤推进，可以避免最常见的"先做视觉、后补内容"的返工陷阱：

<Steps>
  <Step title="明确页面目标与唯一行动号召">
    每个页面只能有一个核心目标——是让用户注册、购买、还是预约？在动手设计之前，先用一句话写清楚：**"我希望访问这个页面的用户做______。"** 目标不清晰，后续所有设计决策都会摇摆不定。
  </Step>

  <Step title="设计首屏（Above the Fold）">
    用户滚动页面之前看到的区域就是首屏。首屏必须在 5 秒内传达三件事：你是谁、你能帮用户解决什么问题、用户下一步应该怎么做。核心元素包括：清晰的主标题（H1）、一句话副标题、以及一个视觉上最突出的 CTA 按钮。

    <Note>
      首屏的背景图或视频会显著影响加载速度。如果你使用背景图，务必将其压缩到 200KB 以内，并使用 WebP 格式。避免在首屏使用自动播放的大体积视频。
    </Note>
  </Step>

  <Step title="建立排版系统">
    在进行任何视觉设计之前，先确定你的字体层级：H1 大小、H2 大小、正文大小、小字注释大小，以及它们对应的行距和字重。正文字号不要小于 **16px**，行距建议在 **1.5–1.8** 之间。整站使用不超过 2 种字体（一种用于标题，一种用于正文）。
  </Step>

  <Step title="定义色彩规范">
    建立一个简单的色彩系统：**主色（Brand Color）** 用于品牌识别，**强调色（Accent Color）** 专门用于 CTA 按钮和重要链接，**中性色** 用于背景和文字。主色不要超过 3 种，强调色只用一种并保持一致性。颜色越克制，视觉层级越清晰。
  </Step>

  <Step title="优化图片与媒体资源">
    所有图片统一转换为 **WebP 格式**，在保持视觉质量的前提下文件体积通常比 JPEG 小 25–35%。每张图片都要填写 `alt` 属性，这既是无障碍要求，也对 SEO 有帮助。使用 [Squoosh](https://squoosh.app/) 或 [TinyPNG](https://tinypng.com/) 进行在线压缩，目标是单张图片不超过 150KB。
  </Step>

  <Step title="设计 CTA 按钮">
    CTA 按钮是页面转化的核心触点。按钮文案要使用动词开头的主动句，例如"立即免费试用"而不是"了解更多"。按钮颜色必须与背景有足够对比度（WCAG AA 标准要求对比度不低于 4.5:1）。在长页面上，每隔一个屏幕左右就应该出现一次 CTA，不要让用户需要滚回顶部才能行动。
  </Step>

  <Step title="检查移动端适配">
    超过 60% 的网页流量来自移动设备。检查以下关键项：所有可点击元素（按钮、链接）的触摸目标不小于 **44×44px**；正文在手机屏幕上单列显示，不出现横向滚动条；图片不超出屏幕宽度。在 Chrome DevTools 里用设备模拟器逐一检查 375px（iPhone SE）和 390px（iPhone 14）断点。
  </Step>

  <Step title="测试页面加载速度">
    用 [PageSpeed Insights](https://pagespeed.web.dev/) 测试你的页面，目标是移动端 Performance 分数达到 **70 分以上**，桌面端 **90 分以上**。重点关注 LCP（最大内容绘制）和 CLS（累积布局偏移）两项指标，这两个指标直接影响用户体验和 Google 排名。
  </Step>
</Steps>

## 排版：让文字更易读

良好的排版不是审美问题，而是降低用户阅读成本的实用决策。

| 排版要素 | 推荐值 | 常见错误 |
|---------|--------|---------|
| 正文字号 | 16–18px | 使用 12–14px 的"精致小字" |
| 行距（line-height） | 1.5–1.8 | 使用默认的 1.2，行与行太拥挤 |
| 段落最大宽度 | 60–75 个字符（约 640px） | 全屏宽度的超长段落 |
| 标题与正文对比 | H1 至少是正文的 2× | 标题和正文大小差别太小 |
| 深色文字对比度 | ≥ 4.5:1（AA 标准） | 浅灰色文字在白色背景上 |

<Note>
  **无障碍提示**：色盲用户约占男性人口的 8%。不要只用颜色来传递信息（例如"红色=错误，绿色=成功"），应同时搭配图标或文字说明。可以用 [Colour Contrast Analyser](https://www.tpgi.com/color-contrast-checker/) 免费检查你的配色方案是否符合无障碍标准。
</Note>

## 色彩使用：克制即是力量

<CardGroup cols={2}>
  <Card title="主色（Brand Color）" icon="palette">
    用于 Logo、导航栏、品牌相关元素。通常只出现在页面顶部和底部，不大面积铺陈，避免视觉疲劳。
  </Card>
  <Card title="强调色（Accent Color）" icon="bullseye">
    专用于 CTA 按钮和重要链接。全站只用一种强调色，让用户的眼睛自动识别"这里可以点击"。
  </Card>
  <Card title="中性色（Neutral）" icon="circle-half-stroke">
    白色、浅灰、深灰构成页面的背景和文字层级。中性色占页面面积的 60–70%，保证整体视觉平静。
  </Card>
  <Card title="功能色（Semantic Color）" icon="triangle-exclamation">
    绿色=成功，红色=错误，黄色=警告。这三种颜色只用于表单反馈、状态提示等功能性场景。
  </Card>
</CardGroup>

## 移动端适配关键断点

```css
/* 移动优先（Mobile First）写法 */

/* 基础样式：手机端（< 768px） */
.container {
  padding: 0 16px;
  font-size: 16px;
}

/* 平板端（768px – 1024px） */
@media (min-width: 768px) {
  .container {
    padding: 0 32px;
    font-size: 17px;
  }
}

/* 桌面端（> 1024px） */
@media (min-width: 1024px) {
  .container {
    max-width: 1200px;
    margin: 0 auto;
    padding: 0 48px;
    font-size: 18px;
  }
}
```

  设计完成后，邀请 3–5 个目标用户做 5 分钟的简单测试：让他们在不说话的情况下尝试完成页面的核心任务（比如"请找到并点击立即购买按钮"）。你会发现很多自己觉得显而易见的设计，对真实用户来说其实并不直观。
</Tip>
