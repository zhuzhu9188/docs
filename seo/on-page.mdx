写完一篇文章只是第一步，页面 SEO 优化才是让它真正被搜索引擎发现和青睐的关键。很多人花了大量时间创作内容，却在 Title Tag 写法、图片命名、内链布局这些"最后一公里"的工作上敷衍了事，白白损失了本可获得的排名。这篇指南帮你把每一个页面优化细节做到位，让你的内容发挥出最大的 SEO 价值。

## Title Tag：页面最重要的 SEO 元素

Title Tag 是出现在搜索结果中的蓝色可点击标题，也是最直接影响排名和点击率的单一页面因素。写好 Title Tag，需要同时满足搜索引擎和真实用户的需求。

**核心原则：**

- **长度控制在 50-60 个字符**（中文约 25-30 个汉字）：超过 60 字符会被 Google 截断，显示为"..."
- **目标关键词尽量靠前**：Google 更重视靠近标题开头的词
- **加入点击诱因**：数字（"7个方法"）、年份（"2024年最新"）、情感词（"完整指南"）能提升点击率
- **每个页面的 Title Tag 必须唯一**：重复的标题会让 Google 困惑，也会造成内部竞争

<Tabs>
  <Tab title="好的示例">
    ✅ `新手 SEO 完整指南：7步打造持续带来流量的网站（2024）`

    ✅ `Google Ads 投放费用详解：如何控制每次点击成本`

    ✅ `WordPress 建站教程：从零开始搭建企业官网`
  </Tab>
  <Tab title="需要改进的示例">
    ❌ `SEO | 我的博客` — 关键词价值低，没有吸引点击的内容

    ❌ `关于我们 - 专注于为企业提供全方位数字营销解决方案服务的专业公司` — 严重超长，会被截断

    ❌ `首页` — 毫无关键词信息
  </Tab>
</Tabs>

## Meta Description：影响点击率的广告文案

Meta Description 不直接影响排名，但它是搜索结果页的"广告语"，直接影响用户是否点击你的链接。点击率（CTR）越高，Google 越倾向于给你更好的排名。

**写作要点：**
- 长度控制在 **130-155 个字符**，过长会被截断
- 包含目标关键词（Google 会加粗显示匹配词）
- 以行动号召结尾，如"立即查看"、"免费下载"、"了解更多"
- 准确概括页面内容，避免诱骗式描述导致高跳出率

<Note>
  如果你不设置 Meta Description，Google 会自动从页面内容中截取片段。自动生成的片段往往不够理想，建议每个重要页面都手动设置。
</Note>

## H1-H6 标题层级的正确使用

标题层级不只是视觉格式，它帮助 Google 理解页面的内容结构和各部分的重要性。

| 标签 | 使用规则 |
|------|---------|
| H1 | 每页只能有一个，包含主要关键词，描述页面核心主题 |
| H2 | 文章主要章节标题，可包含次要关键词和语义相关词 |
| H3 | H2 章节下的子标题，进一步细分内容 |
| H4-H6 | 深层嵌套内容，实际使用较少，避免过度使用 |

**常见错误**：为了视觉效果随意使用 H 标签，导致层级混乱，或同一页面出现多个 H1。

## 内容优化：质量远大于数量

<Accordion title="关键词密度与自然融入">
  关键词密度（关键词出现次数 / 总字数）在现代 SEO 中已不是核心指标，Google 更看重语义相关性。建议做法：
  - 目标关键词在**第一段、H1、至少一个 H2、文章结尾**各出现一次
  - 使用语义相关词（LSI 关键词）填充内容，例如写"SEO 优化"时，自然融入"搜索引擎排名"、"关键词研究"、"自然流量"等相关概念
  - 不要刻意堆砌关键词，读起来不自然的内容反而会被 Google 降权
</Accordion>

<Accordion title="内容深度与竞争对手对标">
  打开你的目标关键词的前3名结果，分析它们的内容结构：
  - 文章字数大概多少？（你的内容不应明显短于竞争对手）
  - 涵盖了哪些子话题？（你需要至少覆盖这些，并争取补充他们遗漏的内容）
  - 有没有使用图表、工具、示例？（丰富的内容格式有助于用户停留）

  优质内容的标志不是字数多，而是真正解答了用户的问题，并且比竞争对手解答得更全面、更清晰。
</Accordion>

<Accordion title="内容新鲜度">
  Google 偏爱定期更新的内容，尤其是对于有时效性的话题（如"2024年最新 SEO 趋势"）。建议：
  - 每隔 6-12 个月对排名页面进行内容审查和更新
  - 更新后修改文章发布日期（如果内容确实有实质性更新）
  - 在文章开头注明"最后更新时间"，提升用户信任感
</Accordion>

## 图片 SEO

图片往往是页面 SEO 中最被忽视的部分，但处理得当可以带来额外的图片搜索流量，并改善页面加载速度。

<Steps>
  <Step title="优化文件名">
    上传图片前，将文件名改为描述性的英文或拼音，用连字符分隔。

    ❌ `IMG_20240315_123456.jpg`

    ✅ `google-ads-tounfang-liucheng.jpg`（Google广告投放流程）
  </Step>
  <Step title="填写 Alt 属性">
    Alt 文本（替代文字）是图片无法加载时显示的描述，也是搜索引擎理解图片内容的主要方式。要求：简洁描述图片内容，自然融入关键词，但不要纯粹堆砌关键词。

    ✅ `alt="Google Ads 广告系列结构示意图"`
  </Step>
  <Step title="压缩图片大小">
    未经压缩的图片是页面加载慢的头号原因之一。建议：
    - 使用 **WebP 格式**代替 JPEG/PNG（同等质量下体积更小）
    - 使用 [Squoosh](https://squoosh.app)（免费在线工具）压缩图片
    - 单张图片大小尽量控制在 **200KB 以内**
  </Step>
  <Step title="添加图片尺寸属性">
    在 HTML 中为 `<img>` 标签明确指定 `width` 和 `height`，可以减少 CLS（累积布局偏移），改善 Core Web Vitals 分数。
  </Step>
</Steps>

## 内链策略

内链（Internal Links）是指你网站内部页面之间相互链接，是构建网站权重流通和帮助用户导航的重要手段。

**内链的两大作用：**
1. **权重传递**：外链带来的"权重"通过内链在站内流通，指向重要页面的内链越多，该页面的权重越高
2. **爬取效率**：内链帮助 Google 爬虫发现和重新爬取你的页面

**最佳实践：**
- **锚文本要有描述性**：用"关键词调研完整指南"而不是"点击这里"作为内链锚文本
- **链接相关页面**：写到某个概念时，链接到你网站中详细介绍该概念的页面
- **重要页面获得更多内链**：你想排名的核心页面，应该从多个其他页面链接过去
- **避免孤立页面**：每个页面至少要有 1-2 个内链指向它，否则爬虫可能忽视它

## URL 优化

<Tip>
  URL 结构一旦确定、已被索引就尽量不要轻易修改，否则需要设置 301 重定向，且修改后排名通常会有短期波动。建站初期就规划好 URL 结构最为稳妥。
</Tip>

**URL 优化原则：**
- **简洁明了**：`/seo/keyword-research/` 优于 `/blog/post?id=12345&cat=3`
- **包含目标关键词**：URL 中的关键词有一定 SEO 价值
- **使用小写字母和连字符**：`/on-page-seo/` 而不是 `/On_Page_SEO/`
- **避免过深的目录层级**：尽量不超过 3 级（`domain.com/category/subcategory/page/`）
- **避免特殊字符和中文**：中文 URL 在分享时会被编码，显示混乱

## 结构化数据（Schema Markup）

结构化数据是一种告诉搜索引擎"这个内容是什么类型"的代码标记，可以让你的搜索结果展示丰富摘要（Rich Snippets），例如星级评分、FAQ 展开、面包屑导航等，显著提升点击率。

<Accordion title="Article Schema — 文章类内容">
  适用于博客文章、新闻报道、教程等内容页面。

  ```json
  {
    "@context": "https://schema.org",
    "@type": "Article",
    "headline": "SEO 基础原理：理解搜索引擎如何发现和排名你的网站",
    "author": {
      "@type": "Person",
      "name": "张明"
    },
    "datePublished": "2024-03-15",
    "dateModified": "2024-06-01",
    "image": "https://example.com/images/seo-guide.jpg",
    "publisher": {
      "@type": "Organization",
      "name": "网站建设心得",
      "logo": {
        "@type": "ImageObject",
        "url": "https://example.com/logo.png"
      }
    }
  }
  ```
</Accordion>

<Accordion title="FAQPage Schema — 常见问题">
  适用于包含问答形式内容的页面，可以在搜索结果中直接展开问答，占据更多屏幕空间。

  ```json
  {
    "@context": "https://schema.org",
    "@type": "FAQPage",
    "mainEntity": [
      {
        "@type": "Question",
        "name": "SEO 需要多久才能看到效果？",
        "acceptedAnswer": {
          "@type": "Answer",
          "text": "一般来说，新网站需要 3-6 个月才能看到明显的自然流量增长。长尾关键词通常比核心关键词更快出现排名。"
        }
      },
      {
        "@type": "Question",
        "name": "SEO 和 SEM 有什么区别？",
        "acceptedAnswer": {
          "@type": "Answer",
          "text": "SEO 是通过优化获得免费自然流量，SEM（搜索引擎营销）通常指付费广告（如 Google Ads）。两者可以结合使用。"
        }
      }
    ]
  }
  ```
</Accordion>

<Accordion title="BreadcrumbList Schema — 面包屑导航">
  让搜索结果中显示页面的层级路径（如"首页 > SEO > 页面优化"），帮助用户了解页面位置，提升点击率。

  ```json
  {
    "@context": "https://schema.org",
    "@type": "BreadcrumbList",
    "itemListElement": [
      {
        "@type": "ListItem",
        "position": 1,
        "name": "首页",
        "item": "https://example.com"
      },
      {
        "@type": "ListItem",
        "position": 2,
        "name": "SEO 指南",
        "item": "https://example.com/seo/"
      },
      {
        "@type": "ListItem",
        "position": 3,
        "name": "页面 SEO 优化",
        "item": "https://example.com/seo/on-page/"
      }
    ]
  }
  ```
</Accordion>

<Note>
  结构化数据添加后，可以使用 [Google Rich Results Test](https://search.google.com/test/rich-results) 验证代码是否正确，并预览在搜索结果中的显示效果。Schema 代码通常放在页面的 `<head>` 标签内，或通过 WordPress 插件（如 Yoast SEO、Rank Math）自动生成。
</Note>
